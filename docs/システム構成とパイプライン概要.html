KawaiiLive システム構成とパイプライン概要

対象: app/, scripts/dev.js, package.json / 最終更新: 2026-06-26

現在の主経路

KawaiiLive は、Control UI から v3 台本を生成し、SQLite に保存し、Irodori TTS で音声化してから、DB 上の音声付き speech line を順番に再生するローカル配信ツール。

npm run dev
  ├─ Ollama 起動確認/起動
  └─ app/server.js
       ├─ app/ui/dist の Control UI 配信
       ├─ Control API
       ├─ app/daihon による v3 台本生成
       ├─ data/talk-items.sqlite への保存
       ├─ app/onair/tts.js による Irodori 音声合成
       └─ app/onair/radio.js による音声再生

ディレクトリ構成

場所責務
app/server.jsHTTP サーバ。API、静的ファイル配信、UI 配信、Ollama/llama-server 起動、OBS 操作、CSV/ZIP 出力をまとめる。
app/onair/配信実行層。台本 plan の DB 保存/読込、音声合成、再生キュー、字幕、OBS、配信状態、utterance state を扱う。
app/daihon/台本生成層。memory/vector を使い、v3 の planning、block 生成、怪談差し込み、品質検査、script item 化を行う。
app/ui/React/Vite の Control UI。ビルド成果物は app/ui/dist に置かれ、app/server.js が配信する。
data/SQLite、vector-db、logs、audio、radio-state などの実行時データ。
assets/背景動画、アバター、sprite、live2d 表示用素材。

対象既定値用途

実行時の前提

  • 再生は DB に保存済みで、かつ speech_lines.audio_path が埋まった行だけを対象にする。
  • 台本生成は app/daihon/generate.mjs を中心にした v3 のみ。
  • 新規実装は app/server.js、app/onair/、app/daihon/、app/ui/ の責務境界に合わせる。
