# KawaiiLive 台本生成仕様 v3.7

最終更新: 2026-06-26  
対象コード: `app/daihon/`, `app/onair/script.js`

## 目的

v3 台本生成は、短い一問一答ではなく、約2分相当の長文ブロックを複数生成して、配信1本分の `speech_lines` として保存する仕組みである。

現行実装では v2 経路は存在しない。台本生成の入口は `app/onair/script.js` の `prepareBroadcastScript()` で、実体は `app/daihon/generate.mjs` の `generateScriptV3()`。

## 入力

`POST /api/script/prepare` から主に以下を受け取る。

| 項目 | 意味 |
| --- | --- |
| `minutes` | 生成したい配信時間。1 block は約120秒として block 数に変換する。 |
| `turns` | block 数を直接指定する場合に使う。 |
| `mode` | DB 保存モード。`create` / `append` / `update`。 |
| `model` | v3 生成に渡すモデル名。未指定なら `RADIO_SCRIPT_OLLAMA_MODEL` または既定値。 |
| `count` | 複数 live を連続生成する batch 数。 |

## block 数

`app/onair/script.js` は以下の基準で block 数を決める。

- `V3_SECONDS_PER_BLOCK = 120`
- `V3_MIN_BLOCKS = 5`
- `effectiveTurns = turns ?? max(5, round(minutes * 60 / 120))`

## 生成の構成

`app/daihon/generate.mjs` は次の順で script item を作る。

1. `memory` を `data/talk-items.sqlite` からランダム取得する。
2. 可能なら怪談 research を読み、OP / MAIN / ED の差し込み候補にする。
3. OP block を生成する。
4. Body loop で director が複数 block の plan を作る。
5. block generator が plan に沿って長文 block を生成する。
6. inspector が品質問題を検査する。
7. block を `script item` に変換する。
8. Ending block を生成する。

## app/daihon の責務

| ファイル | 役割 |
| --- | --- |
| `generate.mjs` | v3 生成のオーケストレーション、LLM 呼び出し、ログ出力。 |
| `constants.mjs` | prompt type、block 長、plan サイズなどの定数。 |
| `planner.mjs` | memory 候補選定、使用済み memory/hook の制御。 |
| `director.mjs` | 次の複数 block の plan 生成。 |
| `block.mjs` | LLM による長文 block 生成。 |
| `parser.mjs` | LLM 出力の整形と block 分割。 |
| `inspector.mjs` | 品質検査。重複、異物、文体崩れなどを確認。 |
| `item-builder.mjs` | block から DB 保存用 script item を作る。 |
| `transition.mjs` / `hooks.mjs` | 遷移 hook 抽出と話題接続の補助。 |
| `vector.mjs` | style/topic/flow の vector 検索。 |
| `kaidan.mjs` / `kaidan-prompts.mjs` | 怪談 research の選択と専用 prompt。 |
| `memory.js` | memory DB の読み込み。 |
| `state.mjs` | stream state の更新。 |

## LLM backend

既定は `LLAMA_SERVER_URL` の OpenAI 互換 `/v1/chat/completions` を使う。`RADIO_SCRIPT_LLM_BACKEND=ollama` の場合は Ollama `/api/chat` を使う。

関連環境変数:

- `LLAMA_SERVER_URL`
- `OLLAMA_URL`
- `RADIO_SCRIPT_OLLAMA_MODEL`
- `RADIO_SCRIPT_LLAMA_TIMEOUT_MS`
- `RADIO_SCRIPT_LLM_BACKEND`
- `RADIO_LOG_LLM`

## 出力 item

生成された item は `app/onair/script.js` に渡され、`speech_lines` に保存される。主な項目は以下。

| 項目 | 意味 |
| --- | --- |
| `text` | 発話本文。 |
| `memoryId` | 元 memory。怪談系など memory に紐づかない場合は null 相当。 |
| `anchor` | 話題の接続語/核。 |
| `generator_req` / `generator_res` | block generator の prompt/response。 |
| `director_req` / `director_res` | director の prompt/response。 |
| `transition_type` / `transition_keyword` / `transition_reason` | 実際の遷移情報。 |
| `prompt_type` | `OPENING`, `DEEP_DIVE`, `RESET_TRANSITION`, `ENDING`, `KAIDAN_*` など。 |
| `source_type` | `memory` / `kaidan` など。 |
| `research_*` | 怪談 research の識別情報。 |
| `handoff_*` | 前 block からの受け渡し情報。 |
| `selected_plan_block` | 採用された director plan。 |

## DB 保存

`app/onair/script.js` は `savePlanToDb()` で以下を行う。

- `live` を新規作成、または append 対象 live を選ぶ。
- 必要な `speech_lines` カラムを `alter table` で補う。
- script item を `sequence_no` 付きで `speech_lines` に挿入する。
- 最新 plan mirror として `data/logs/script-plan.json` も更新する。

## 品質方針

- 1行の短文ではなく、配信でそのまま読めるまとまった block にする。
- memory の具体物、行動、感情の手触りを保つ。
- 連続 block は唐突に切らず、hook と handoff でつなぐ。
- 抽象語だけの感想、異言語混入、定型句、同一 memory の使いすぎを避ける。
- 怪談 research は配信全体の流れに挿入し、生活雑談と断絶しないように扱う。

## ログ

v3 の LLM ログは `data/logs/script-llm-v3.jsonl` に追記される。Control UI の status では末尾ログを `scriptLlmLog` として返す。
