# KawaiiLive 台本生成評価観点

最終更新: 2026-06-26  
対象: `app/daihon/` で生成され、`data/talk-items.sqlite` の `live` / `speech_lines` に保存された v3 台本。

## 評価の目的

生成された live が、配信でそのまま読めるか、記憶素材を活かしているか、話題遷移が自然かを確認する。評価値は `POST /api/live/eval` から `live` テーブルへ保存できる。

## 評価項目

| 項目 | DB カラム | 重み | 見ること |
| --- | --- | ---: | --- |
| 深さ | `eval_depth` | 30% | memory の具体物や状況が掘れているか。薄い一般論で終わっていないか。 |
| 流れ | `eval_flow` | 25% | block 間の接続が自然か。唐突な reset が続いていないか。 |
| 自然さ | `eval_naturalness` | 20% | 話し言葉として読めるか。説明文、論文調、過度な定型句になっていないか。 |
| 根拠性 | `eval_grounding` | 15% | memory / research / vector 由来の内容から逸脱していないか。 |
| 反復少なさ | `eval_repetition` | 10% | 同じ言い回し、同じ memory、同じ展開が続いていないか。 |

全項目が入っている場合、重み付き合計は `app/server.js` 側で算出される。

## 確認ポイント

- `speech_lines.sequence_no` が連続している。
- `text` が空でない。
- memory 由来の行は `memory_id` が入っている。
- 怪談由来の行は `source_type` や `research_*` で追える。
- `generator_req/res`、`director_req/res` が必要に応じて保存されている。
- `audio_path` は合成前なら空、合成後は wav path の JSON 配列。
- 再生後は `status` と `spoken_at` が更新される。

## 品質 NG 例

- 1 block が短すぎて配信の間が持たない。
- 「さて」「では」「今日は」だけで話題が切り替わり、前 block の hook を拾っていない。
- memory にない固有名詞や出来事を断定する。
- 怪談 research が突然挿入され、生活雑談との接続がない。
- 同じ表現、同じ物、同じ感想が数 block 続く。
- LLM のメタ発言、推論タグ、英語混入、不要な記号が残る。

## 確認コマンド

```powershell
node --check app/server.js
node --check app/onair/radio.js
node --check app/onair/script.js
node --check app/daihon/generate.mjs
```

UI 側を確認する場合:

```powershell
npm.cmd run build
```

実行ディレクトリは `app/ui`。

## 運用メモ

- 仕様上、台本生成と音声合成と再生は別段階。
- 音声が未合成の live は `Radio Start` でエラーになる。
- `script-plan.json` は確認用 mirror。正は SQLite。
- v3 の調整はまず `app/daihon/` の prompt、director、inspector、item builder のどこに効かせるかを切り分ける。
