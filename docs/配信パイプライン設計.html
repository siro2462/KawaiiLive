KawaiiLive 配信パイプライン設計

Control Server / daihon / onair / UI / 最終更新: 2026-06-26

標準操作フロー

1. npm run dev
2. http://127.0.0.1:14520/ を開く
3. Script Prepare を実行
4. live / speech_lines が data/talk-items.sqlite に保存される
5. Synthesize を実行
6. speech_lines.audio_path に wav 配列が入る
7. Radio Start で DB plan を読み、音声付き speech line を再生
8. 必要に応じて Stop、Emergency Stop、CSV 出力、OBS 操作を行う

主要 API

API用途
GET /api/statusOllama、radio、broadcast、OBS、script LLM log の状態。
GET /api/liveslive 一覧。総行数と音声合成済み行数を含む。
GET /api/script?liveId=...指定 live または最新の DB plan を返す。
GET /api/memoriesmemory 一覧。
GET /api/sqlite, /api/sqlite/tableSQLite カタログとテーブル閲覧。
GET /api/live/lines, /api/live/audiospeech line と wav 取得。
GET /api/live/csv, /api/live/csv-all単体 live CSV と全体 ZIP 出力。
POST /api/script/preparev3 台本生成。minutes, count, mode, model を受ける。
POST /api/script/cancel台本生成または音声合成の abort。
POST /api/live/synthesize未合成 speech line を Irodori で wav 化。
POST /api/radio/start指定 live の音声付き DB plan を再生開始。
POST /api/radio/stop安全停止。
POST /api/radio/emergency-stop即時停止。
POST /api/live/delete, DELETE /api/livelive と speech_lines、および対象 audio ディレクトリを削除。
POST /api/live/evallive 評価値を保存。
POST /api/obs/*OBS 接続、切断、配信開始/停止、シーン切替。
GET /api/vector/*style/topic/flow vector DB の検索と閲覧。
POST /api/server/restart, /api/server/stopControl Server 再起動、関連プロセス停止。
POST /api/logs/clearscript LLM log と runtime talk log をクリア。

RadioRuntime

app/onair/radio.js は DB plan を再生キューへ積み、wav を PowerShell の Media.SoundPlayer で同期再生する。字幕は wav 長と文単位分割から推定して currentText に反映する。

状態意味
stopped停止中。台本生成と音声合成を開始可能。
startingDB plan と runtime state を読み、キューを準備中。
runningproduce loop と play loop が動作中。
stopping安全停止要求済み。
error音声欠落、再生失敗、DB 読込失敗など。

音声合成

app/onair/tts.js の IrodoriWorkerClient が uv run --extra {backend} python tts_worker.py を library/irodori で起動する。既定値は IRODORI_BACKEND=cu128、IRODORI_DURATION_SCALE=1.05、IRODORI_NUM_STEPS=24。

静的アセット

app/server.js は UI 以外に、avatar、sprite、live2d、背景動画をローカル配信する。背景動画一覧は /api/background-videos から取得し、実体は assets/backgrounds/onair に置く。
