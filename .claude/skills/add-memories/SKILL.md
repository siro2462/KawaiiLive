# add-memories — 断片記憶の生成と追加

## 手順

1. **種の準備**: ユーザーから種（1行メモ）をもらう。なければ以下から10〜20個自作する:
   - 季節・時期（今日の日付から: 夏なら冷房・花火・アイス）
   - ターゲット台本（assets/vtuber台本/）のトピック型の翻案（内容コピーはNG、「布教」「企画振り返り」等の型だけ）
   - 既存 memory の category 偏りを見て少ない系統を補う
   - 種は「状況+具体物」まで書く（「コンビニの話」ではなく「雨上がりの夜にコンビニでビニール傘を2本目買った」）

2. **生成**: 種1件ごとに下のプロンプトの思考手順に従い、自分で（このセッションのモデルで）JSONを生成する

3. **検証**: 各件について — 習慣形（「〜することがある」「よく〜する」）が混じっていないか / 数字が最低1つあるか / 五感の断片があるか / 既存 memory と keywords が被っていないか（被ったら別の切り口に修正）

4. **投入**: `scripts/add-memories.mjs` があればそれを使う。なければ node ワンライナーで直接 insert:
   - `insert into memory (keywords, episode) values (?, ?)` — keywords は JSON文字列（例 `["コンビニ","傘"]`）
   - `episode_v2` カラムは存在するので同時に入れる。`category` 等の拡張カラムは `pragma table_info(memory)` で存在確認してから（B6 未実施なら episode/episode_v2/keywords だけ入れ、拡張フィールドはJSONごと `data/memory-source/YYYYMMDD.json` に保存しておく）

5. **報告**: 追加件数・categoryの内訳・保存先

## 生成プロンプト（この思考手順に従う）

```
あなたは配信者「ZIPちゃん」の記憶を作る。目的は、本人しか知らない一回きりの断片記憶。
実在の人物・配信者の出来事は使わない。店名・商品名・値段は実在のものを使ってよい。

手順（1〜4を考えてから、最後にJSONだけを出力する）:
1. この出来事が起きた「一回」を決める: 曜日・時間帯・天気・直前に何をしていたか
2. 断片ディテールを8個列挙する: 値段/個数/棚の位置/音/匂い/手の動き/他の客/店員の様子
3. そのうち「本筋と関係ないのに妙に覚えていること」を1つ選ぶ
4. 「思い出せない部分」を1つ決める（なんでそうしたのか・その後どうしたか等）

出力形式:
{
  "keywords": ["…", "…"],
  "episode": "…",          // 一回きりの出来事として3〜5文。数字を最低1つ、五感の断片を最低1つ
  "episode_v2": "…",       // 同じ内容をZIPちゃんの一人称の話し言葉で3〜5文
  "category": "food|game|fail|discovery|outing|goods|health|season",
  "mood": "布教|笑い話|しみじみ|グチ|発見",
  "people": [],
  "punchline": "…",
  "objects": ["…"],        // 位置・状態つきの具体物 2〜4個
  "actions": ["…"],        // 小さい行動 2〜3個
  "micro_conflicts": ["…"],
  "odd_detail": "…",       // 本筋と関係ないのに覚えていること
  "unclear": "…"           // 思い出せない部分
}

条件:
- 習慣形（「〜することがある」）は禁止。その日一回の出来事として書く
- 抽象語（感覚・違和感・空気感・現実感）で説明しない。物と行動と数字で書く
- きれいなオチをつけすぎない。人の記憶は途中で終わってよい
- 事件にしない。生活の小さい話に留める
- episode_v2 は「〜なんだよね」「〜しちゃってさ」等の配信口調。顔文字・記号は使わない
```

良い例・悪い例は docs/ロードマップ.html「データ整備手順」の memory生成プロンプト参照。

## 既存455件の書き直しモード

ユーザーが「既存の書き直し」を求めたら: episode を種として同じプロンプトに渡し、習慣形→一回きりの断片記憶に変換 + episode_v2 未生成分の補完 + 拡張フィールド生成を一度にやる。update文で episode_v2（と存在する拡張カラム）だけ更新し、**episode 原文は上書きしない**（元データ保全）。
